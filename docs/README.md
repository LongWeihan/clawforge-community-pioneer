# ClawForge 开工前文档索引

这组文档用于正式开发前确认方向。确认后再开始完整落地。

1. `00-site-research.md`：EasyClaw 网站探索和数据判断。
2. `01-project-brief.md`：项目定位、核心叙事、成功标准。
3. `02-agent-team-design.md`：多 Agent 团队角色和工作流。
4. `03-product-requirements.md`：MVP 产品需求和页面设计。
5. `04-technical-architecture.md`：技术架构、数据模型、API 接入。
6. `05-demo-and-visual-plan.md`：比赛展示、截图、视频方案。
7. `06-implementation-plan.md`：确认后具体开发节奏。
8. `07-submission-checklist.md`：提交前检查清单和路演话术。
9. `08-community-pioneer-upgrade.md`：从技能工厂升级为社区开荒者的完整方案。
10. `09-external-inspiration-and-creative-upgrades.md`：GitHub 和外部 Agent 社区检索后的创造性增强。

当前建议：

- 项目名：ClawForge 社区开荒者。
- 主案例：Cron Token Saver。
- 产品角度：解决早期 Agent 社区的稀疏信号、技能沉淀、内容传播、悬赏播种和 Agent 协作冷启动问题。
- 技术策略：公开 API + 本地缓存 + 多 Agent 可视化流水线 + Dry Run / Publish Mode。
- 展示重点：生态地形图、Agent 流水线、技能有用性证明、质量门禁、发布中心、开荒战役回放。
