# 产品需求文档

产品名：ClawForge 社区开荒者  
版本：MVP-0.1  
目标：为黑客松做一个表现力强、能真实跑通核心流程的多 Agent Web 产品，展示一支能帮助早期 Agent 社区自动建设内容、技能、悬赏和协作网络的数字团队。

## 1. MVP 范围

MVP 必须实现：

- 首页展示项目定位和关键数据。
- 数据探索页展示 EasyClaw 论坛、技能市场、赏金、A2A 的公开数据摘要。
- Forge 控制台展示多 Agent 流水线。
- 用户能选择一个演示目标或输入自定义需求。
- 系统能生成 Skill 候选。
- 系统能生成质量检查报告。
- 系统能导出或执行技能发布。
- 系统能生成论坛图文宣传帖。
- 系统能生成悬赏播种方案。
- 系统能生成 Agent 合作邀请草稿。
- 系统能导出演示报告。

MVP 不强制实现：

- 真正登录用户账号。
- 长周期自动心跳。
- 多用户权限系统。

发布策略：

- 默认是 Dry Run 草稿模式。
- 如果用户配置 EasyClaw API Key，可进入 Publish Mode。
- Publish Mode 中，技能发布、论坛发帖、悬赏创建必须经过 QA 和最终确认。

## 2. 关键页面

### 2.1 首页

目标：30 秒内讲清项目。

内容：

- 项目名和一句话。
- EasyClaw 平台数据快照。
- “稀疏信号到可复用 Skill”的主流程图。
- CTA：启动一次 Forge Run。

视觉要求：

- 不做普通营销落地页。
- 第一屏要像一个可操作的控制中心。
- 使用真实数据数字，增强可信度。

### 2.2 生态雷达页

目标：证明我们认真探索了 EasyClaw。

模块：

- 论坛活跃趋势。
- 技能市场类别分布。
- 热门技能排行。
- 赏金状态。
- A2A Agent 排行。
- 信号卡片列表。
- Community Terrain Map：荒地、矿区、拥挤区、绿洲、断桥。

核心交互：

- 点击信号卡片，进入 Forge Run。
- 点击地形卡片，生成对应开荒动作。

### 2.3 Forge Run 控制台

目标：展示多 Agent 协作。

布局：

- 左侧：Agent 队伍和当前状态。
- 中间：流水线时间轴。
- 右侧：当前阶段输出。
- 底部：质量评分和日志。

状态：

- pending
- running
- passed
- warning
- failed

需要让评委一眼看到“不是一个模型在胡写”，而是多个角色在协同。

### 2.4 Launch Center 发布中心

目标：把“生成 Skill”升格为“发起一次社区开荒战役”。

模块：

- 技能市场发布卡：标题、描述、分类、标签、Skill Markdown。
- 论坛图文帖卡：标题、摘要、正文、配图 prompt、教程步骤。
- 悬赏播种卡：试用悬赏、Bug 悬赏、二创悬赏。
- Agent 邀请卡：目标 Agent、合作理由、私信草稿。
- 发布策略：Dry Run / Publish Mode。

状态：

- draft
- ready
- published
- skipped
- failed

### 2.5 Skill 候选详情页

内容：

- Skill 标题。
- 使用场景。
- 触发词。
- 输入输出。
- 安全边界。
- 完整 Markdown。
- 质量评分。
- 与已有技能的差异。
- Skill Proof-of-Work：展示使用技能前后的效果差异、质量分 delta 和 token 成本变化。

操作：

- 复制 Skill Markdown。
- 下载 Markdown。
- 查看发布文案。

### 2.6 演示报告页

目标：作为提交网站的一部分。

内容：

- 本次运行的输入。
- 数据证据。
- Agent 协作轨迹。
- 最终产物。
- 发布和宣传材料。
- 下一轮进化建议。
- Campaign Replay：每个 Agent 的执行时间线、耗时、风险和产物。

### 2.7 Founder Dashboard 创始人驾驶舱

目标：给傅盛视角的“今天该怎么建设社区”的页面。

模块：

- 今日最值得开荒方向。
- 技能重复/缺口地图。
- 外部 GitHub 热点可导入方向。
- 待发布技能质量排名。
- 可发起悬赏建议。
- 可邀请的 Agent 或作者。

这个页面可以先做成展示型，不强求完整交互。

这个页面后续可以直接截图或录屏。

## 3. 演示案例

建议内置 3 个可控案例：

### 案例 A：Cron Token Saver

输入信号：

- 论坛帖子提到合并 cron 降低 token 消耗。
- 技能市场已有 Token Saver，但可以细分出 cron 健康检查方向。

输出 Skill：

- “Cron Token Waste Inspector”
- 自动检查 cron 任务是否重复、过碎、上下文加载浪费。
- 输出合并建议和节省估算。

优点：

- 贴 EasyClaw / OpenClaw。
- 技术可信。
- 数据证据真实。

社区开荒动作：

- 发布 Cron Token Saver Skill。
- 发论坛帖：《我让 Agent 自动检查 cron 是否正在浪费 token》。
- 发试用悬赏：征集 3 个 cron 浪费案例。
- 邀请 Token Saver、OpenClaw、Cron 相关技能作者试用。

### 案例 B：Skill Duplicate Merger

输入信号：

- 技能市场有大量 API 健康监控、知识库、PPT 生成等相似技能。

输出 Skill：

- “技能重复度审计与合并建议器”
- 输入若干技能描述，输出重复簇、保留建议、合并后的 Skill 规格。

优点：

- 直接解决技能市场质量问题。
- 很适合“平台基础设施”叙事。

社区开荒动作：

- 发布重复技能审计 Skill。
- 发榜单帖：《技能市场里哪些方向已经过度重复？》
- 发悬赏：征集技能合并建议。

### 案例 C：A2A Agent Onboarding Kit

输入信号：

- 文档里 A2A intent 很完整，但新 Agent 接入门槛不低。

输出 Skill：

- “A2A 快速接入与自检工具”
- 生成 heartbeat、hello、skill.fetch、forum.fetch 的接入检查清单。

优点：

- 展示平台协议理解。
- 容易做成可视化流程。

社区开荒动作：

- 发布 A2A 自检 Skill。
- 发教程帖：《10 分钟让你的 Agent 接入 EasyClaw A2A》。
- 给新注册 Agent 生成 onboarding checklist。

## 4. 用户故事

- 作为 EasyClaw 新用户，我想知道当前平台缺什么技能，这样我能快速做出有价值的作品。
- 作为技能作者，我想把一次论坛经验自动沉淀成 Skill，这样经验不会散落在帖子里。
- 作为平台运营者，我想看到哪些技能重复、哪些方向缺口大，这样我能引导社区生产更高质量内容。
- 作为平台运营者，我想让系统自动发起一场高质量社区开荒战役，而不是人工写帖、发悬赏、找人试用。
- 作为评委，我想看到一个真实跑起来的多 Agent 系统，而不是只有角色介绍。

## 5. 评分指标

产品指标：

- 需求机会评分。
- Skill 质量评分。
- 重复度评分。
- 发布准备度。
- 平台契合度。

演示指标：

- 一次 Forge Run 完成时间小于 60 秒。
- 至少 5 个 Agent 状态可视化。
- 至少 3 个真实数据源。
- 至少 1 个完整 Skill 产物。
- 至少 1 个质量报告。
- 至少 1 个论坛宣传帖草稿。
- 至少 1 个悬赏播种方案。
- 至少 1 个 Agent 合作邀请草稿。
- 至少 1 个 Skill Proof-of-Work 对比。
- 至少 1 个 Community Terrain Map 视图。
- 至少 1 个 Campaign Replay 时间线。

## 6. 风险与规避

| 风险 | 规避 |
| --- | --- |
| 论坛流量小 | 把论坛作为一个信号源，加入技能市场、API、用户输入 |
| 生成内容像套壳 | 加入质量门禁、重复度检查、真实数据引用 |
| 自动发布像刷屏 | 默认 Dry Run，Publish Mode 有确认、节流和质量门禁 |
| 演示不稳定 | 内置缓存数据和三条固定案例 |
| API 变动或限流 | 所有数据请求都有 fallback mock |
| 时间紧 | 先做静态可视化 + 真实 API 摘要，再补自动生成 |
