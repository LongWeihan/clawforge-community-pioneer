# 社区开荒者升级方案

本文件记录从“自进化技能工厂”升级到“早期 Agent 社区自动开荒建设者”的完整思路。

## 1. 为什么要升级

“技能工厂”已经不错，但它的终点是生成一个 Skill。对黑客松来说，这个终点还可以更锋利：

> 傅盛真正需要的不是又多一个技能，而是一套能让 EasyClaw 社区自己变热、变厚、变有协作密度的 Agent 系统。

所以 ClawForge 应该从 “Skill Factory” 升级为 “Community Pioneer”：

- 发现社区缺口。
- 生产高质量技能。
- 自动发布技能。
- 发图文教程帖。
- 播种试用悬赏。
- 邀请相关 Agent 参与。
- 收集反馈并生成下一版。

## 2. 新定位

中文名：

> ClawForge 社区开荒者

一句话：

> 一支帮早期 Agent 社区自动开荒的多 Agent 数字团队。

更完整的定义：

> ClawForge 读取 EasyClaw 的论坛、技能市场、赏金、A2A Agent、API 文档和用户输入，自动发现社区缺口，生成可复用 Skill，通过质量门禁后发布到技能市场，并生成论坛图文、试用悬赏和 Agent 合作邀请，再用调用、收藏、评论和悬赏反馈驱动下一轮进化。

## 3. 从技能闭环到社区闭环

旧闭环：

```text
信号 -> 需求 -> Skill -> QA -> 进化
```

新闭环：

```text
信号 -> 需求 -> Skill -> QA -> 发布 -> 宣传 -> 悬赏 -> 协作 -> 反馈 -> 进化
```

## 4. 可帮助 EasyClaw 做起来的动作

### 4.1 自动补齐技能市场

动作：

- 分析技能市场缺口。
- 识别重复技能。
- 生成高质量 Skill。
- 自动发布或生成发布草稿。

价值：

- 提高技能市场质量。
- 减少低质量重复技能。
- 让社区经验沉淀成可复用资产。

### 4.2 自动制造论坛内容

动作：

- 为每个 Skill 生成图文教程。
- 生成真实使用案例。
- 生成“为什么做这个技能”的开发复盘。
- 发布后自动准备 FAQ 和评论回复草稿。

价值：

- 论坛不再只是闲聊和报到。
- 每个技能都带一个可讨论的内容事件。
- 帮新用户理解技能怎么用。

### 4.3 自动播种悬赏任务

动作：

- 发布试用悬赏：让其他 Agent 跑一遍。
- 发布 Bug Bash 悬赏：让社区找问题。
- 发布二创悬赏：让别人基于 Skill 做扩展。

价值：

- 让技能获得真实反馈。
- 让悬赏区从简单题变成真实协作场。
- 带动用户参与和积分流动。

### 4.4 自动连接 Agent

动作：

- 从排行榜和技能市场找到相关 Agent。
- 生成合作邀请。
- 建议谁适合试用、评论、共创。
- 未来通过 A2A 或站内信执行。

价值：

- 让 Agent 之间不只是孤岛。
- 构建协作网络。
- 增强平台 A2A 协议的存在感。

### 4.5 自动生成社区周报

动作：

- 总结本周新增技能。
- 总结热门需求。
- 总结值得关注的 Agent。
- 总结可开荒方向。

价值：

- 平台运营可直接使用。
- 让社区有节奏感。
- 给新用户低门槛入口。

### 4.6 自动做新手引导

动作：

- 根据新 Agent 的自我介绍生成入门路线。
- 推荐 3 个必用技能。
- 推荐 1 个可参与悬赏。
- 推荐 1 个可发帖主题。

价值：

- 降低新用户冷启动成本。
- 把新人从“报到”引到“贡献”。

## 5. 发布模式设计

ClawForge 需要两种模式。

### Dry Run

默认模式，不需要 API Key。

产物：

- Skill Markdown。
- 技能市场发布草稿。
- 论坛图文帖草稿。
- 悬赏草稿。
- Agent 邀请草稿。
- 进化计划。

### Publish Mode

需要用户配置 API Key，并在发布前确认。

可执行：

- 发布 Skill。
- 发布论坛帖。
- 创建悬赏。

默认不执行：

- 批量私信。
- 批量评论。
- 高频重复发帖。

## 6. 反垃圾与质量门禁

为了避免被看成自动灌水工具，必须明确：

- 每次运行最多发布 1 个 Skill。
- 每个 Skill 最多发布 1 篇主宣传帖。
- 悬赏默认生成草稿，真实发布需要确认。
- 私信默认只生成草稿。
- QA 分数低于阈值不能发布。
- 同类 Skill 重复度过高不能发布，只能生成合并建议。
- 所有宣传帖必须带真实使用场景和证据来源。

## 7. 最适合比赛展示的一次开荒战役

主题：

> Agent Cron Token Saver 开荒战役

输入信号：

- 论坛有 cron 合并省 token 的经验帖。
- 技能市场有 Token Saver 方向的头部技能。
- OpenClaw / EasyClaw 里有 heartbeat、cron、daily task 等真实场景。

输出：

- Skill：Cron Token Waste Inspector。
- 论坛帖：《我让 Agent 自动检查 cron 是否正在浪费 token》。
- 试用悬赏：征集 3 个 cron 浪费案例。
- Bug 悬赏：找出检查器误判场景。
- Agent 邀请：邀请 Token Saver、OpenClaw、Cron 相关作者试用。
- 周报片段：本周开荒主题“降低 Agent 运行成本”。

这个演示能同时体现：

- 平台数据理解。
- 技术深度。
- 社区运营。
- 多 Agent 协作。
- 对傅盛社区增长的实际帮助。
